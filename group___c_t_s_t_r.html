Macro Definition Documentation

#define CTSTR_MAX_STRING_SIZE   0x100

Defines the largest string size allowed.

Set this to whatever you want, as long as it's a power of 2 and <= 0x10000. Whatever you set it to, just be aware that the larger the number is the more your compiler will hate you.

#define CTSTR_SEQUENCE_TYPE   std::integer_sequence

Allows an alternative to std::integer_sequence for pre C++14 compilers.

By default this macro is just std::integer_sequence. If you want to use something else, or your standard library doesn't provide std::integer_sequence, just #define CTSTR_SEQUENCE_TYPE to be your own custom type. Make sure it has the same signature as std::integer_sequence, though.

#define string_to_type (   str)

Turns a quoted "string" into a unique type.

Creates an std::integer_sequence containing the characters that make up the string.

template<typename SeqType >
auto ctstr::to_c_string ( SeqType  sequence = SeqType{})

Turns a string returned from string_to_type into a C-style "string".

This is just like to_string() except the returned type is a pointer to a sequence of characters. You do not need to delete or free this memory. It exists as a static array.

Under most circumstances you'll probably just want to stick to to_string(). But if you have a function that just takes a const char* and want to save the memory allocation and calling str.c_str() and whatnot, this works too.

template<typename SeqType >
auto ctstr::to_string ( SeqType  sequence = SeqType{})

Turns a string returned from string_to_type into an std::string.

You can either use to_string<string_to_type("foo")>() or to_string(string_to_type("foo"){}). Pick whichever you find to be neater.

The only other thing of note is that it's actually an std::basic_string with the char type the type used in the std::integer_sequence you created. Besides that, it's literally just creating a string that contains your characters.
